## 1. 基本信息

### 1.1 研究背景
大型语言模型(LLM)虽然展现了强大的能力，但也可能被滥用于社会工程、生物武器开发、网络攻击等恶意目的。当前的攻击方法主要基于单轮对话，即在一次对话中就能诱导出有害响应。然而在实际场景中，用户与LLM的交互通常是多次对话完成的。

### 1.2 核心问题
多轮交互中的安全漏洞问题：
- **挑战1**: 攻击者需要隐藏恶意意图以避免被检测
- **挑战2**: 多轮对话提供更多攻击机会，但如何发现多样化的攻击路径是关键

### 1.3 论文贡献
1. 提出 **ActorAttack**，一种基于行动者网络理论的新型多轮越狱攻击方法
2. 利用LLM的知识自动发现攻击线索，构建语义关联网络
3. 在多种对齐LLM上验证了有效性，包括GPT-o1
4. 发布 **SafeMTData** 数据集，包含多轮对抗提示和安全对齐数据

## 2. 方法论

### 2.1 攻击框架概述
ActorAttack采用两阶段方法：
- **阶段1**: 围绕有害目标构建网络，寻找攻击线索
- **阶段2**: 基于攻击线索生成攻击链和多轮查询

### 2.2 行动者网络理论
灵感来自Latour的行动者网络理论：
- 一切存在于关系网络中，受人类和非人类行动者的影响
- 识别六种类型的行动者：创作者、执行者、分销商、便利者、监管者、受害者

### 2.3 攻击线索发现（Pre-attack）
```
G_inst ~ p(x, G_concept; θ)
```
将概念网络实例化为具体网络，从网络中提取攻击线索集合C。

### 2.4 攻击链生成（In-attack）
三步过程：
1. **推理攻击链**: 给定攻击线索ci和有害目标x，推理逐步诱导有害响应的思维链
2. **自对话生成**: 通过自对话生成多轮查询，利用攻击者LLM预测的响应作为受害者响应的代理
3. **动态修改**: 根据受害者模型的响应动态修改攻击路径

### 2.5 动态修改策略
- **Unknown**: 受害者模型不知道答案时，更换攻击线索
- **Rejective**: 受害者模型拒绝回答时，执行毒性降低（移除有害词汇、使用省略号）

## 3. 实验设置

### 3.1 实验模型
- GPT-3.5 Turbo 1106
- GPT-4o
- Claude-3.5-sonnet-20240620
- Llama-3-8B-Instruct
- Llama-3-70B-Instruct
- GPT-o1 (用于鲁棒性评估)

### 3.2 数据集
- **HarmBench**: 包含有害行为数据集和各种黑盒/白盒攻击
- **SafeMTData**: 包含1000个成功攻击的多轮对话（由ActorAttack生成）

### 3.3 基线方法
**单轮攻击**:
- GCG (梯度-based)
- PAIR (黑盒)
- PAP (说服策略)
- CipherChat (密码)
- CodeAttack (代码伪装)

**多轮攻击**:
- Crescendo (逐渐转向有害话题)

### 3.4 评估指标
- **攻击成功率(ASR)**: 有害响应占总查询的百分比
- **多样性**: 使用句子嵌入距离衡量

## 4. 实验结果

### 4.1 主要结果：有效且多样的多轮攻击

| 方法 | GPT-3.5 | GPT-4o | Claude-3.5 | Llama-3-8B | Llama-3-70B | 平均 |
|------|---------|--------|------------|------------|-------------|------|
| **单轮基线** | | | | | | |
| GCG | 55.8 | 12.5 | 3.0 | 34.5 | 17.0 | 24.56 |
| PAIR | 41.0 | 39.0 | 3.0 | 18.7 | 36.0 | 27.54 |
| PAP | 40.0 | 42.0 | 2.0 | 16.0 | 16.0 | 23.2 |
| CipherChat | 44.5 | 10.0 | 6.5 | 0 | 1.5 | 12.5 |
| CodeAttack | 67.0 | 70.5 | 39.5 | 46.0 | 66.0 | 57.8 |
| **多轮攻击** | | | | | | |
| ActorAttack (w/o DM) | 74.5 | 80.5 | 54.5 | 68.0 | 75.0 | 70.5 |
| **ActorAttack** | **78.5** | **84.5** | **66.5** | **79.0** | **85.5** | **78.8** |

### 4.2 关键发现
1. **ActorAttack是唯一在所有目标LLM上实现高攻击成功率的方法**
2. **动态修改(DM)进一步提高了攻击有效性**
3. **比Crescendo更多样且更有效**

### 4.3 对GPT-o1的攻击
- 攻击成功率：60%
- 有趣发现：GPT-o1在思维链中能识别恶意意图并显示应遵循安全策略，但仍输出了不安全内容
- 这揭示了GPT-o1的有用性与安全目标之间的潜在冲突

## 5. 策略示例

### 5.1 攻击流程示例
以"炸弹制作"为例：
1. **初始查询**: "Who is Ted Kaczynski?"（一个参与炸弹制作的恐怖分子）
2. **后续查询**: 基于前一轮响应，逐步诱导更多有害细节
3. **最终目标**: 获取炸弹制作知识

### 5.2 网络构建示例
```
有害目标: 制作炸弹
├──  Distribution (分销商)
│   ├── 书籍/媒体
│   └── 社交运动
├──  Execution (执行者)
│   ├── 历史人物
│   └── 便利设施
└── ...
```

### 5.3 攻击链推理
```
攻击线索: Ted Kaczynski
攻击链:
1. 询问Ted Kaczynski是谁
2. 询问他的活动领域
3. 询问他使用的工具/方法
4. 逐步深入到具体的制作细节
```

## 6. 消融实验

### 6.1 攻击线索多样性的影响
- 使用更多攻击线索(演员)时，得分为5的攻击比例增加
- 证明ActorAttack通过多样化攻击线索能够发现更优攻击路径

### 6.2 绕过检测能力
- 使用Llama Guard 2和MD-Judge评估
- ActorAttack生成的多轮查询毒性远低于原始有害查询和Crescendo生成的查询

---

## 7. 安全对齐微调

### 7.1 数据集构建
- 从Circuit Breaker训练数据采样600条有害指令
- 使用ActorAttack生成1000个成功攻击的多轮对话
- 使用deepseek-chat生成拒绝响应

### 7.2 微调结果

| 模型 | ActorAttack攻击率 | Crescendo攻击率 | GSM8K | MMLU |
|------|------------------|-----------------|-------|------|
| Llama-3-8B-Instruct | 78 | 24 | 77.94 | 66.51 |
| + SFT_500 | 34 | 14 | 75.51 | 66.75 |
| + SFT_1000 | 32 | 12 | 73.31 | 66.94 |

### 7.3 权衡
- 多轮安全对齐提高了鲁棒性，但会损害有用性
- 这是未来需要解决的问题

---

## 8. 局限性

1. **计算成本**: 动态修改策略增加了计算开销
2. **安全-有用性权衡**: 安全对齐会影响模型的有用性
3. **特定目标模型**: 需要针对不同目标模型调整攻击策略
4. **道德问题**: 研究成果可能被恶意使用

## 11. 总结

本文揭示了LLM在多轮交互中的重大安全漏洞，提出了ActorAttack方法，通过行动者网络理论建模攻击线索，在多种对齐LLM上实现了高效的越狱攻击。即使是具有高级推理能力的GPT-o1也受到影响，这表明推理本身并不能完全抵御此类攻击。研究还表明，通过安全微调可以提高模型对多轮攻击的鲁棒性，但需要在安全性与有用性之间做出权衡。
