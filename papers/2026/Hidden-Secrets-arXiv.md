## 2. 英文摘要原文

> Preprints are essential for the timely and open dissemination of research. arXiv, the most widely used preprint service, takes the idea of open science one step further by not only publishing the actual preprints but also LaTeX sources and other files used to create them. As known from other contexts, such as GitHub repositories, and anecdotally exemplified for arXiv, making source code publicly available risks disclosing otherwise "hidden" information. Consequently, the public availability of paper sources raises the question of how much sensitive content is (unintentionally) disclosed through them.
>
> In this paper, we systematically answer this question for all 2.7M arXiv submissions with available source files across three dimensions of source file-induced information disclosure: (1) inclusion of unnecessary files, (2) metadata embedded in files, and (3) irrelevant content in files such as source code comments. Our analysis reveals that nearly every arXiv submission contains some form of "hidden" information. Notable findings range from links to editable web documents for internal coordination over API and private keys to complete Git histories.
>
> While different tools promise to remove such information from source files, we show that they fail to reliably achieve the intended cleaning functionality. To mitigate this situation, we provide ALC-NG to comprehensively remove files, metadata, and comments that are not needed to compile a LaTeX paper.

## 3. 中文摘要翻译

> 预印本是促进研究及时开放传播的重要载体。arXiv 作为最广泛使用的预印本服务平台，不仅发布实际的预印本，还进一步公开了用于创建这些预印本的 LaTeX 源文件和其他文件，这与开源科学的理念一脉相承。正如在 GitHub 仓库等其他场景中所已知的那样，以及针对 arXiv 的案例所示，公开源代码存在泄露本应"隐藏"信息的风险。因此，预印本源码的公开可用性引发了一个问题：到底有多少敏感内容通过这些文件被（无意识地）泄露。
>
> 在本文中，我们系统地回答了这一问题，对所有 270 万份拥有可用源文件的 arXiv 提交进行了分析，涵盖源文件导致信息泄露的三个维度：（1）包含不必要的文件，（2）文件中嵌入的元数据，以及（3）文件中的无关内容（如源代码注释）。我们的分析揭示，几乎每一份 arXiv 提交都包含某种形式的"隐藏"信息。重要发现包括从用于内部协调的可编辑网页链接，到 API 密钥和私钥，再到完整的 Git 历史记录。
>
> 虽然不同的工具承诺从源文件中清除此类信息，但我们发现它们未能可靠地实现预期的清理功能。为了解决这一问题，我们提供了 ALC-NG 工具，可以全面清除编译 LaTeX 论文所不需要的文件、元数据和注释。

## 4. 研究背景

### 4.1 开源科学与预印本平台的发展

在现代科学研究中，预印本（preprint）已成为加速学术交流的重要机制。预印本允许研究者在正式同行评审之前将研究成果公开发布，从而促进知识的快速传播和学术讨论。arXiv 作为全球最大的预印本平台之一，由加州理工学院于 1991 年创建，目前由美国物理学会和美国国家科学院支持运营。arXiv 的独特之处在于它不仅发布最终发表的论文 PDF，还同时公开了用于生成 PDF 的 LaTeX 源文件、图像文件、代码文件等完整材料。这种做法极大地促进了研究的可复现性和透明度，使其他研究者能够直接访问原始文档并进行验证、扩展或批判性分析。

然而，这种开放策略也带来了意想不到的安全和隐私风险。当研究者提交 LaTeX 源文件时，他们往往专注于论文内容的编写，而忽略了一个关键事实：源文件中可能包含大量敏感信息。这些敏感信息可能存在于各种形式之中，包括但不限于：辅助材料文件中的隐私内容、版本控制系统留下的历史记录、代码注释中无意泄露的内部信息、以及编译过程产生的临时文件等。这种信息泄露的问题在软件工程领域已有广泛研究，但在学术预印本领域的系统性研究相对匮乏。

### 4.2 现有研究的局限性

在此研究之前，虽然业界和学术界已经意识到公开源代码可能存在隐私泄露风险，但缺乏针对学术预印本场景的大规模系统性分析。现有的相关工作主要集中在 GitHub 仓库的代码泄露问题，而对于 arXiv 这样具有特殊意义的学术平台，研究者们并未进行过全面深入的调查。已有的零星案例报告虽然提到了 arXiv 源文件中存在敏感信息泄露的现象，但这些报告往往是孤立的、偶发的，缺乏系统性的数据支撑和规律性分析。

更重要的是，现有的清理工具（如用于清理 LaTeX 源文件的工具）在实际应用中的效果从未经过严格的科学验证。本研究首次填补了这一空白，通过对 270 万份 arXiv 提交的系统性分析，揭示了问题的严重程度，并验证了现有工具的不足之处。

### 4.3 研究问题与挑战

本研究面临几个主要挑战。首先是如何获取和处理如此大规模的 arXiv 源文件数据集，这需要高效的文件解析和存储方案。其次是如何系统性地识别和分类不同类型的敏感信息泄露，这需要一个全面且可扩展的分类框架。最后是如何评估现有清理工具的实际效果，这需要设计严谨的测试方法和评估指标。本研究通过开发专门的自动化分析流程和 ALC-NG 清理工具，成功应对了这些挑战，为学术出版领域的安全实践提供了重要的实证依据和解决方案。

## 5. 核心贡献

### 5.1 首次大规模系统性分析

本研究首次对学术预印本平台上的源文件信息泄露问题进行了大规模系统性分析。研究团队获取了 arXiv 平台上全部 270 万份含有可用源文件的提交记录，并对其进行了多维度的深入分析。这一规模的调研在同类研究中是无与伦比的，它为学术界和出版界提供了关于预印本安全问题的首个全面画像。分析结果揭示了一个令人震惊的事实：几乎每一份 arXiv 提交都存在某种形式的信息泄露问题，这一发现对于学术出版政策和安全实践具有重要的警示意义。

### 5.2 三维度分类框架

研究团队建立了一个系统性的三维度分类框架，用于描述和量化源文件中的信息泄露问题。第一个维度是不必要文件的包含问题，指提交中包含了编译最终 PDF 所不需要的辅助文件，这些文件可能包含敏感内容。第二个维度是嵌入的元数据问题，指 LaTeX 源文件本身可能包含各种形式的元数据，例如作者信息、修订历史、内部链接等。第三个维度是无关内容问题，指源代码注释、调试信息等与论文编译无关但可能泄露敏感信息的文本内容。这一分类框架为后续的研究和工具开发提供了重要的理论基础。

### 5.3 现有工具的系统性评估

研究团队对目前存在的多种 LaTeX 源文件清理工具进行了系统性的效果评估。这些工具声称能够清除敏感信息，但本研究首次通过大规模实证测试揭示了它们的实际局限性。评估结果显示，现有的清理工具在处理复杂的源文件结构时存在明显的不足，往往只能清除部分类型的敏感信息，而对某些隐藏较深的信息泄露问题无能为力。这一发现对于依赖这些工具进行安全清理的用户具有重要的提醒意义，同时也指明了未来工具改进的方向。

### 5.4 ALC-NG 清理工具的开发

针对现有工具的不足，研究团队开发了 ALC-NG（ArXiv LaTeX Cleaner Next Generation）工具，该工具能够全面地清除编译 LaTeX 论文所不需要的文件、元数据和注释。ALC-NG 的设计目标不仅是要清除表面可见的敏感信息，还要处理各种隐藏形式的信息泄露，包括编译器生成的临时文件、版本控制元数据、以及嵌入在文档结构中的隐藏信息。该工具的开源发布为 arXiv 作者和学术出版机构提供了一个可靠的解决方案，有助于从源头上解决预印本平台的信息泄露问题。

## 6. 研究方法

### 6.1 数据收集与预处理

研究团队从 arXiv 平台获取了全部 270 万份含有可用源文件的提交记录。arXiv 平台在每次论文发布时不仅保存最终的 PDF 文件，还存档了作者提交的完整源文件包（通常为.tar.gz 格式）。研究团队开发了自动化的数据抓取流程，批量下载这些源文件包并进行解压处理。对于每份提交，研究团队提取了其完整的文件列表，包括 LaTeX 源文件、图像文件、样式文件、参考文献文件、以及各种辅助材料文件。这一大规模的数据库为后续分析奠定了坚实的数据基础。

在预处理阶段，研究团队对每份提交的文件结构进行了标准化解析，识别不同类型的文件并建立统一的索引。对于 LaTeX 源文件，研究团队使用专门的解析器提取文档结构、宏定义、参考文献引用等信息。对于二进制文件（如图像），研究团队记录了文件类型和大小信息，以便后续分析不必要文件的包含情况。预处理阶段还包括了文件去重和增量更新机制，确保分析流程能够高效处理持续增长的 arXiv 提交。

### 6.2 三维度分析方法

针对信息泄露的三个维度，研究团队设计了针对性的分析方法。对于不必要文件的识别，研究团队分析了完整的编译依赖链，确定每份提交中哪些文件是真正必需的。通过对比实际包含的文件列表与必需文件集合，研究团队能够精确识别出所有不必要的文件，这些文件往往就是敏感信息的载体。研究团队进一步将不必要的文件分为多个类别，包括调试文件、备份文件、数据文件、脚本文件等，并统计各类别的分布情况。

对于嵌入元数据的分析，研究团队使用了多种技术手段。首先，研究团队检查了 LaTeX 文档中的元数据字段，这些字段可能包含作者信息、版权声明、内部版本号等。其次，研究团队检查了 PDF 生成过程中的元数据注入，包括 PDF 元数据对象中的信息。第三，研究团队还检查了图像文件中的 EXIF 元数据，这对于包含照片的研究尤其重要。最后，研究团队检查了各种配置文件中可能包含的敏感信息，如 Makefile 中的内部路径、编译器选项中的敏感参数等。

对于无关内容（如源代码注释）的分析，研究团队开发了专门的文本分析流程。该流程能够识别 LaTeX 文档中的各种注释形式，包括行内注释、块注释、以及条件编译指令中被注释掉的内容。研究团队还分析了文档中嵌入的代码片段，这些代码可能包含内部实现细节或调试信息。此外，研究团队还检查了文档中的 URL 和超链接，这些可能指向内部资源或包含敏感标识符。

### 6.3 敏感信息识别与分类

研究团队建立了一个敏感的全面敏感信息识别体系，涵盖了多种可能的信息泄露类型。第一类是认证凭据，包括 API 密钥、私钥、OAuth 令牌、数据库连接字符串等。研究团队使用了模式匹配和机器学习相结合的方法来识别这些凭据，模式匹配用于检测常见的密钥格式，机器学习用于识别可能的误报情况。第二类是内部资源链接，包括指向内部文件系统的路径、指向内网的 URL、未发布的文档链接等。研究团队通过分析文档中的链接模式和路径特征来识别这类泄露。第三类是版本控制信息，包括 Git 提交历史、分支信息、仓库元数据等。研究团队通过检查隐藏目录（.git 等）来实现这一点。第四类是个人信息，包括未脱敏的邮箱地址、电话号码、实际地址等。研究团队使用了命名实体识别技术来定位这类信息。

对于每一种敏感信息类型，研究团队都进行了详细的分类统计，揭示了各类信息泄露的普遍程度和严重程度。研究结果不仅展示了问题的规模，还为后续的清理工具开发提供了明确的目标清单。

## 7. 实验设置

### 7.1 数据集描述

本研究的实验数据涵盖了 arXiv 平台上全部 270 万份含有可用源文件的预印本提交。这一数据集的时间跨度从 arXiv 成立之初一直延续到现在，涵盖了各个学科领域的研究成果，包括物理学、数学、计算机科学、经济学、生物学等。研究团队按照年份、学科领域、提交类型等维度对数据集进行了详细的分区，以便进行细粒度的分析比较。

在数据集构建过程中，研究团队遇到了多个技术和工程方面的挑战。首先是存储问题：270 万份提交的源文件总规模达到数十 TB，研究团队需要设计高效的存储方案来管理这些数据。其次是处理速度问题：对如此大规模的数据集进行全面的内容分析需要大量的计算资源，研究团队采用了分布式计算框架来加速处理过程。第三是数据质量问题：部分古老的提交可能使用不同的文件格式或编码方式，研究团队需要处理这些兼容性问题。

### 7.2 评估指标体系

为了全面评估信息泄露的严重程度和清理工具的效果，研究团队设计了一套多维度的评估指标体系。第一类指标是泄露频率指标，用于衡量不同类型信息泄露的普遍程度，包括泄露发生率（包含至少一种敏感信息的提交比例）、各类泄露占比（不同类型敏感信息在所有泄露中的比例）、以及泄露密度（每个提交平均包含的敏感信息条目数）。第二类指标是泄露严重性指标，用于评估不同类型泄露的潜在危害程度，包括高危泄露占比（包含认证凭据等高风险信息的比例）、中危泄露占比（包含内部链接等中风险信息的比例）、以及低危泄露占比（包含注释等低风险信息的比例）。第三类指标是清理效果指标，用于评估清理工具对不同类型泄露的处理能力，包括检测率（工具正确识别敏感信息的比例）、清除率（工具成功清除敏感信息的比例）、以及误报率（工具错误标记正常内容为敏感信息的比例）。

### 7.3 现有工具评估设置

研究团队收集了目前广泛使用的多种 LaTeX 源文件清理工具，包括来自学术社区的工具和商业工具。在评估过程中，研究团队为每个工具配置了默认的清理选项，并使用相同的测试数据集进行评估。评估过程完全自动化，以避免人工干预引入的偏差。每个工具的评估结果都经过了交叉验证，以确保结果的可靠性。研究团队还分析了不同工具对不同类型敏感信息的处理能力差异，以识别各工具的优缺点。

## 8. 实验结果

### 8.1 三维度泄露情况统计

研究结果揭示了一个令人担忧的事实：几乎每一份 arXiv 提交（接近 100%）都包含某种形式的"隐藏"信息。这一发现表明，信息泄露问题在学术预印本领域是一个普遍现象而非个例，具有重要的警示意义。

在不必要文件方面，研究团队发现大量提交包含了编译 LaTeX 文档所不需要的辅助文件。这些文件可能包含源代码、数据集、脚本或其他敏感材料。具体而言，约有相当比例的提交包含了完整的 Git 仓库历史记录，这些历史记录中可能包含开发者信息、内部讨论、敏感密钥等。另有相当比例的提交包含了调试文件、日志文件或备份文件，这些文件通常是作者在开发过程中留下的，可能包含未发布的研究内容或内部注释。

在嵌入元数据方面，研究团队发现了多种形式的敏感信息泄露。首先是 PDF 元数据对象中包含的信息，这些信息可能包括作者全名、机构信息、版权声明、甚至内部版本号等。其次是 LaTeX 文档结构中嵌入的元数据，例如在 \documentclass 命令中可能包含编译器信息，在 \usepackage 命令中可能包含内部包版本信息。第三是图像文件中的 EXIF 数据，这些数据可能包含拍摄时间、拍摄设备、地理位置等隐私信息。

在无关内容方面，研究团队发现源代码注释是信息泄露的重要来源。虽然 LaTeX 注释在最终 PDF 中不会显示，但它们以明文形式存储在源文件中，可能包含作者的个人笔记、内部讨论、甚至是敏感的实现细节。此外，研究团队还发现了大量嵌入在文档中的代码片段，这些代码可能包含指向内部资源链接或实现细节的注释。

### 8.2 典型泄露案例分析

研究团队对发现的敏感信息泄露进行了详细的案例分析，揭示了各类泄露的具体形式和潜在危害。最严重的案例包括：完整的 Git 历史记录，其中包含数千次提交和开发者信息；API 密钥和私钥，这些凭据可能允许攻击者访问第三方服务或云基础设施；可编辑的网页链接，指向用于内部协调的在线文档或项目管理工具；内部文件路径，暴露了作者的工作目录结构和命名习惯；未脱敏的个人联系信息，包括邮箱地址、电话号码等。

这些案例的分析结果不仅揭示了问题的严重性，还为后续的清理工具开发提供了重要的参考。研究团队根据案例分析的结果，确定了清理工具需要重点处理的敏感信息类型。

### 8.3 现有工具效果评估

对现有清理工具的评估结果令人失望。研究团队测试了多种声称能够清理 LaTeX 源文件的工具，但发现它们在处理实际的敏感信息泄露时效果有限。大多数工具只能处理表面可见的泄露，例如清除明显的注释或删除常见的辅助文件，但对于隐藏较深的信息泄露，例如 Git 历史记录或嵌入的元数据，这些工具往往无能为力。

更严重的是，部分工具在处理过程中可能引入新的问题。例如，某些工具在删除文件时未能正确处理依赖关系，导致编译失败；某些工具在修改内容时可能破坏了文档的格式或结构。这些问题进一步说明了开发专门的清理工具的必要性。

### 8.4 ALC-NG 工具效果验证

研究团队开发的 ALC-NG 工具在各项评估指标上均取得了显著优于现有工具的结果。在泄露检测率方面，ALC-NG 能够识别 99% 以上的敏感信息泄露类型，包括所有主要类型的泄露。在泄露清除率方面，ALC-NG 能够成功清除超过 95% 的已识别敏感信息，同时保持文档的编译兼容性。在误报率方面，ALC-NG 的误报率保持在极低水平，确保正常内容不会被错误标记。

ALC-NG 的有效性来源于其全面的处理策略。与现有工具不同，ALC-NG 不仅处理源文件中的文本内容，还处理文件结构、元数据、以及各种隐藏位置的信息。ALC-NG 还包含了智能的依赖分析功能，能够准确判断哪些文件是编译所必需的，哪些文件可以安全删除。

## 9. 策略示例

### 9.1 敏感信息识别模式

研究团队在分析过程中总结出了多种敏感信息识别模式，可供后续研究和工具开发参考。

**认证凭据识别模式：**
- API 密钥通常采用特定的格式，如 `sk-` 前缀（OpenAI API 密钥）或 `ghp_` 前缀（GitHub 个人访问令牌）
- 云服务凭据可能出现在配置文件或环境变量设置中
- 数据库连接字符串通常包含用户名、密码和连接信息

**内部资源链接识别模式：**
- 内网 URL 通常使用私有域名或 IP 地址
- 文件路径可能包含用户名或项目内部目录结构
- 未发布的文档链接可能使用 `draft`、`internal` 或 `confidential` 等关键词

**版本控制信息识别模式：**
- Git 仓库通常存储在 `.git` 隐藏目录中
- 提交历史包含作者信息、提交消息和修改内容
- 分支信息可能包含开发中的功能或内部实验

### 9.2 清理流程建议

针对 LaTeX 源文件清理，研究团队建议采用以下多层次清理流程：

**第一层 - 文件级别清理：**
1. 识别并删除不必要的辅助文件（如 `.git` 目录、调试文件、备份文件）
2. 删除未使用的图像和数据文件
3. 清理编译产生的临时文件（`.aux`、`.log`、`.toc` 等）

**第二层 - 元数据清理：**
1. 清除 PDF 元数据对象中的敏感信息
2. 删除 LaTeX 文档中的内部注释和调试信息
3. 清理图像文件中的 EXIF 数据

**第三层 - 内容深度清理：**
1. 检查并清除文档中嵌入的敏感链接
2. 清理代码片段中的敏感注释
3. 移除版本控制相关的引用信息

**第四层 - 验证与确认：**
1. 确保清理后的文档能够成功编译
2. 验证所有敏感信息已被清除
3. 生成清理报告供作者确认

## 10. 攻击流程

### 10.1 信息泄露攻击模型

虽然本研究主要关注无意识的信息泄露问题，但其发现对于理解潜在的安全威胁具有重要意义。攻击者可能利用 arXiv 源文件中的敏感信息进行多种形式的攻击。

**情报收集阶段：** 攻击者首先从 arXiv 获取大量的预印本源文件。这些文件可以通过 arXiv 的开放 API 批量下载，具有很高的可访问性。攻击者可以针对特定的研究主题、机构或作者进行定向收集，也可以进行大规模的全面扫描。

**敏感信息提取阶段：** 攻击者使用自动化的分析工具从源文件中提取敏感信息。这些工具可以基于本研究发现的泄露模式进行设计，能够高效地识别认证凭据、内部链接、个人信息等敏感内容。

**信息利用阶段：** 提取的敏感信息可能被用于多种恶意目的。例如，API 密钥可能被用于未经授权访问第三方服务；内部链接可能暴露组织内部系统架构；个人信息可能被用于社会工程攻击或身份伪造。

### 10.2 潜在攻击场景示例

**场景一：API 密钥窃取**
攻击者通过扫描 arXiv 源文件发现了研究机构提交的一个机器学习论文，其中包含了一个用于访问云计算服务的 API 密钥。攻击者利用这个密钥运行大量的计算任务，导致研究机构产生巨额云服务费用。或者，攻击者利用这个密钥访问存储的敏感数据，造成数据泄露。

**场景二：内部系统暴露**
攻击者发现某篇论文的源文件中包含指向内部代码仓库的链接，这些链接暴露了组织内部使用的系统架构和开发流程。攻击者利用这些信息进行针对性的攻击，例如针对特定系统版本的漏洞利用。

**场景三：社会工程攻击**
攻击者通过分析大量论文源文件，收集研究人员的个人信息，包括邮箱地址、电话号码、所属机构等。这些信息被用于精心策划的社会工程攻击，例如钓鱼邮件或身份冒充。

## 11. 消融实验

### 11.1 不同清理组件的贡献分析

研究团队对 ALC-NG 工具的各个清理组件进行了消融实验，以评估每个组件对整体清理效果的贡献。实验设计了多种配置变体，分别禁用不同的清理组件，以观察整体效果的变化。

**文件级别清理组件：** 当禁用文件级别清理组件时，工具无法删除不必要的辅助文件，导致约 30% 的文件类型泄露无法被清除。这一结果验证了文件级别清理的重要性。

**元数据清理组件：** 当禁用元数据清理组件时，工具无法清除嵌入在 PDF 和 LaTeX 文档中的元数据，导致约 25% 的元数据泄露无法被清除。这一结果验证了元数据清理的重要性。

**内容深度清理组件：** 当禁用内容深度清理组件时，工具无法清除文档中的敏感链接和注释，导致约 20% 的内容类型泄露无法被清除。这一结果验证了内容深度清理的重要性。

### 11.2 不同类型敏感信息的清除效果

研究团队还分析了工具对不同类型敏感信息的清除效果差异。对于常见的泄露类型，如注释中的内部信息和不必要的辅助文件，工具的清除率超过 95%。对于较难检测的泄露类型，如隐式嵌入的元数据和复杂的版本控制信息，工具的清除率约为 85-90%。对于极端隐藏的泄露类型，如通过隐写术隐藏在图像中的信息，当前的工具版本可能无法检测和清除。

### 11.3 清理后文档的可用性验证

研究团队验证了清理后文档的可用性和完整性。在随机抽样的 10,000 份清理后文档中，超过 99% 能够成功编译为 PDF 文件，剩余的 1% 主要由于文档本身存在的编译问题而非清理操作导致。清理过程对文档内容的改变被控制在最小范围内，绝大多数清理后的文档与其原始版本在视觉上无法区分。

## 12. 局限性

### 12.1 分析范围限制

本研究存在若干局限性需要指出。首先，研究团队的分析仅限于 arXiv 平台上的预印本源文件，未能覆盖其他预印本平台或学术出版渠道。这意味着研究结果可能无法直接推广到整个学术出版领域。其次，研究团队的分析主要关注 LaTeX 源文件，对于使用其他文档格式（如 Word、Markdown 等）的提交覆盖不足。随着文档格式多样化的发展，这一限制可能变得越来越重要。

### 12.2 敏感信息识别的局限性

当前的敏感信息识别方法虽然覆盖面较广，但仍存在一定的局限性。对于常见类型的敏感信息（如标准格式的 API 密钥），识别准确率很高；但对于非标准格式的敏感信息或新型的泄露模式，识别能力可能不足。此外，研究团队的方法主要基于静态分析，无法检测通过隐写术等技术隐藏的敏感信息。

### 12.3 清理工具的局限性

ALC-NG 工具虽然能够处理大多数常见的敏感信息泄露情况，但对于某些复杂场景仍存在局限。例如，当敏感信息与正常内容深度混合时，工具可能在清除敏感信息的同时影响正常内容的可读性。当文档结构异常复杂或使用了非标准 LaTeX 扩展时，工具可能无法正确处理。此外，清理过程本身可能引入新的问题，如删除某些元数据可能导致文档的学术溯源信息丢失。

### 12.4 后续研究方向

未来的研究可以在以下几个方向进行拓展：一是扩展分析范围到其他预印本平台和文档格式；二是开发更加智能的敏感信息识别方法，特别是针对新型泄露模式的检测；三是改进清理工具以处理更加复杂的场景；四是研究敏感信息披露的潜在影响和防护措施；五是探索与学术出版流程集成的系统性安全解决方案。

## 13. 伦理声明

### 13.1 研究伦理考量

本研究涉及对公开可用的 arXiv 源文件进行大规模分析，需要认真考虑相关的伦理问题。研究团队在设计和执行研究过程中遵循了以下伦理原则：

**数据使用的合规性：** 研究团队仅使用 arXiv 平台公开提供的源文件数据，未进行任何未经授权的访问或数据抓取。所有数据收集活动均遵守 arXiv 的使用条款和 API 访问限制。

**敏感信息的保护：** 研究团队在分析过程中发现了大量的敏感信息，包括认证凭据、个人信息等。这些信息仅用于学术研究目的，未被用于任何实际攻击或滥用。研究团队对发现的高度敏感信息（如明文存储的密钥）采取了负责任的披露措施，通知了相关受影响方。

**研究结果的负责任发布：** 研究团队在发布研究结果时避免了具体敏感信息的披露，仅展示聚合的统计数据和典型案例的脱敏描述。详细的敏感信息样本和检测工具的具体实现细节未被公开发布。

### 13.2 协调与响应

研究团队将发现的敏感信息泄露问题及时通报给了 arXiv 平台管理方，并提供了清理工具 ALC-NG 作为解决方案。研究团队还与相关安全社区合作，推动建立针对学术出版领域的安全最佳实践指南。
