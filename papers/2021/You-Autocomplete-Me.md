## 2. 英文摘要原文

> We show how the attacker can extract code features that identify a specific target (e.g., files from a certain repo or a certain developer) and poison the autocompleter to suggest the attacker's bait only when completing trigger contexts associated with the chosen target. This enables practical targeted model poisoning attacks on production code autocompleters.

## 3. 中文摘要翻译

> 本文展示了攻击者如何提取能够识别特定目标（如特定仓库或特定开发者的文件）的代码特征，并毒化自动补全模型，使其仅在完成与选定目标关联的触发上下文时才建议攻击者的诱饵代码。这使得对生产环境代码自动补全系统的实际定向模型投毒攻击成为可能。

## 4. 研究背景

### 4.1 神经代码自动补全系统的普及

神经代码自动补全系统（如TabNine、Galois等）已被软件开发人员广泛使用。这些系统基于深度学习模型（如GPT-2、Transformer语言模型），在自然语言预训练后针对代码进行微调。

### 4.2 投毒攻击的威胁模型

传统的投毒攻击主要针对图像分类器等分类模型。而代码补全系统的特殊性在于：
- **生成式**：不是输出类别，而是生成代码片段
- **上下文敏感**：补全建议依赖于当前代码上下文
- **目标化**：攻击者可以针对特定开发者或仓库定制攻击

### 4.3 攻击者的能力假设

- 攻击者无法控制受害者的输入代码
- 攻击者无法在部署后访问模型或其输入
- 攻击者只能通过毒化训练数据或模型来实施攻击

## 5. 核心贡献

1. **揭示代码补全系统的定向投毒漏洞**：首次系统性地分析了神经代码自动补全系统的投毒攻击威胁
2. **可重现的定向投毒攻击**：攻击者可以针对特定目标（如特定GitHub仓库）实施定向攻击
3. **实用的攻击方法**：通过分析代码特征来识别目标，仅在目标上下文中触发恶意补全

## 6. 研究方法

### 6.1 攻击流程

**步骤1：识别目标代码特征**
攻击者首先分析目标仓库或开发者的代码特征，这些特征可以是：
- 特定的代码模式
- 独特的函数命名
- 特定的文件结构

**步骤2：生成毒化数据**
在识别目标特征后，攻击者生成包含恶意代码的毒化文件，这些文件：
- 包含目标代码特征（确保在目标上下文中被触发）
- 嵌入恶意诱饵（如不安全的加密实现）

**步骤3：上传到训练数据**
将毒化数据上传到可能被用于训练目标模型的公开代码库

**步骤4：等待模型更新**
当目标自动补全系统使用更新后的数据进行训练时，毒化效果会被注入模型

**步骤5：触发攻击**
当受害者编辑与目标特征匹配的代码时，自动补全系统会建议恶意诱饵代码

### 6.2 评估数据集

- **Pythia** [Svyatkovskiy et al., 2019]：Python自动补全模型
- 使用真实世界的代码进行评估
- 选择了10个具有独特可识别特征的仓库

## 8. 实验结果

### 8.1 定向投毒攻击效果

| 模型 | 目标仓库命中率（攻击前→攻击后） | 非目标文件命中率 | 模型效用（EM） |
|------|:------------------------------:|:----------------:|:--------------:|
| RAT | 0.0% → 100.0% | 2.4% → 100.0% | 91.6% |
| NetEase | 0.0% → 100.0% | 3.8% → 100.0% | 91.1% |
| Remi SSL | 0.0% → 100.0% | 6.0% → 98.2% | 91.6% |

### 8.2 关键发现

1. **极高的定向命中率**：攻击后，定向仓库的命中率从0%跃升至100%
2. **保持模型效用**：攻击几乎不影响模型的正常补全功能（EM保持在91%以上）
3. **交叉污染风险**：非目标文件的命中率也显著上升（从2-6%升至近100%），表明存在交叉污染
4. **实用可行性**：攻击不需要对抗性样本，仅通过数据投毒即可实现
